前端规范 // 代码规范

前端代码规范基于W3C、苹果开发者 等官方文档，并结合团队日常业务需求以及团队在日常开发过程中总结提炼出的经验而制定。 旨在增强团队开发协作、提高代码质量和打造开发基石的编码规范， 以下规范是团队基本约定的内容，必须严格遵循。

  • HTML规范

    基于W3C、苹果开发者等官方文档，并结合团队日常业务需求以及团队在日常开发过程中总结提炼出的经验而约定。

  • 图片规范

    了解各种图片格式特性，根据特性制定图片规范，包括但不限于图片的质量约定、图片引入方式、图片合并处理等。

  • CSS规范

    统一团队 CSS 代码书写和 SASS的语法风格，提供媒体查询语句和浏览器私有属性引用，从业务层面统一规范常用模块的引用。

  • 命名规范

    从 “目录命名”、“图片命名”、“ClassName” 命名等层面约定规范团队的命名习惯，增强团队代码的可读性。

  • JavaScript 规范

    统一团队的 JS 语法风格和书写习惯，减少程序出错的概率，其中也包含了 ES6 的语法规范和最佳实践。

  • 前端解决方案

    提供通用PC、移动端模板，模板包括：基本模板、GULP、Webpack、Vue....

团队 // 技术岗位介绍

前端工程师

1. 3年以上相关工作经验，计算机相关专业，本科以上学历； 2. 精通 JavaScript，HTML5，CSS3，熟悉常见跨浏览器问题，了解必要的计算机网络协议 3. 熟悉主流JavaScript框架和库，了解框架设计原理，能独立开发常用组件 4. 有良好的web前端优化经验，熟悉常用布局以及浏览器和移动设备兼容性 5. 熟悉W3C标准，对表现与数据分离、Web语义化等有深刻理解 6. 具有至少一门服务器端编程的实战经验 7. 熟悉常用设计模式和至少一种MVC框架 8. 对web产品设计和用户体验有一定的了解 9. 优秀的逻辑思维能力和业务需求分析能力，良好的沟通能力，迅速融入团队 10. 参与开源社区项目并贡献过代码者优先
